✅ ISSUE 3: 404 Error After Document Upload
----------------------------------------------------------

PROBLEM:
- After uploading documents, user got 404 error
- Page tried to navigate to /profile#documents
- Used window.location.href causing full page reload
- Hash navigation not working properly

ROOT CAUSE:
- OfferRide.tsx used: window.location.href = '/profile#documents'
- This causes full page reload
- React Router doesn't handle hash properly on reload
- Results in 404 error

FIXED:
✅ Changed from: window.location.href = '/profile#documents'
✅ Changed to: navigate('/profile?tab=documents')
✅ Uses React Router navigation (no page reload)
✅ Uses query parameter instead of hash
✅ Profile page already handles ?tab=documents parameter

RESULT:
✅ No more 404 errors after document upload
✅ Smooth navigation to documents tab
✅ No page reload
✅ Better user experience
